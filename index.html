Here we will examine a simple function, and analyze its syntax. Underscores here are used to indicate indentation, but would need to be replaced with spaces to create a working function.

 

OurFunction <- function(inputargs){

__outputargs <- inputargs*2

}

 

Functions are pretty simple. We would call this function in this way:

 

foo = OurFunction(2)

foo = 4

 

Looking at our original function there are 3 main parts.

1) There is the function name (in this case "OurFunction")

2) There is the input variables (in this case the "function(inputargs)")

3) Finally you have the operations that are performed in the function. Which are held withing braces. It's important to note that operations that are performed within a function are indented (in this case represented by 2 underscores)

2) Common Syntax

BackHomeNext

This section will demonstrate some of the basic syntax of R.

 

If you're coming into R after leanring another language some of this syntax may be familiar to you, or at the least make sense logically.

 

Defining and handling variables is handled slightly differnently than in other languages. R uses something other than the 'equals' sign in object assignment. Instead of ***=*** R uses ***<-*** which logically makes sense because it yeilds

something like this ...

***a <- 3***

Which can be read as '3' is being assigned to 'a' and the directional arrow helps in clarifying exactly what is happening.

 

functions are defined in this way:

FunctionName <- function(inputvariable){

operationsperformed = inputvariable

return(operationsperformed)

}

 

Loops and if statements share similar syntax, one (non-working) example:

LoopType(Loopconditional){

somethingbeingdoneintheloop

}

 

 

1) Intro

BackHomeNext

R is a programming language that is primarally used by scientists and engineers. It is incedebly widely used, and is a very handy language to know. R shares familiar syntax to many languages and also provides some unique elements that intuitive,

and make sense. All of this combined makes R one of the most powerful languages in the world.

 

R is an open-source language that has a very large and helpful community, making getting started with R even easier. Due to its popularity in academia it also has a vast array of litterature written on and around it.

 

R is also home to a large number of open-source libraries that make many common and not-so-common tasks incredibley easy.

 

For the programmer who has multiple languages under his/her belt, it may be helpful to think of R syntax as a little bit of MATLAB combined with at little bit of C/C++. If migrating from MATLAB (which many scientists may be) it is important to note that R lacks some of the work-arounds for simple coding errors that MATLAB offers (for example, you have to initialize matrices in R when MATLAB simply extends the matrix if you're adding a number where one currently doesn't exist).
